Verdict: **ITERATE**.

The Phase 5 package cannot pass. The current final artifacts are too thin to function as a standalone analysis note, contain unresolved plot/rendering red flags, and still assign result-like prominence to an optimized calibrated-score model that fails observed validation. The validated visible-mass baseline may be the sole final result only if the output metadata, AN, PRL draft, figures, captions, and comparison text consistently treat it as the current final result and demote the optimized classifier to a failed diagnostic method study.

This does not automatically require a Phase 3/4 regression if the fixer chooses the baseline-only final-result path. It does require changing current result metadata and document-generation inputs so that the failed calibrated-score workspace is no longer labeled `primary` or used in headline comparisons. If the analysis team instead wants the optimized classifier to be the final result, that requires upstream Phase 3/4 regression: repair the score-model data/MC disagreement, document at least three remediation attempts for the observed validation failure, regenerate the statistical outputs, and re-review.

## Final-Result Role Decision

The optimized calibrated-score model may remain in the analysis note only as a diagnostic method study that failed the observed validation gate. It must not appear in the abstract, conclusion, headline result table, final comparison figures, or final signal-strength/significance comparison as "this analysis" except clearly labeled as a rejected optimization attempt.

The validated visible-mass baseline may become the sole final result if the fixer does all of the following:

1. Make `visible_mass_qcd_primary` the sole final-result model in machine-readable Phase 5 result metadata and document-generation inputs.
2. Demote `calibrated_score_qcd_primary` to `diagnostic_failed_validation` or equivalent in every JSON summary, caption, table, and figure label that is part of the final deliverable.
3. Rewrite the AN and PRL draft so the abstract, result section, comparison section, and conclusion lead with the baseline observed and expected CLs limits and low-sensitivity interpretation.
4. Add full baseline-specific validation, nuisance, impact, injection, GoF, and systematic documentation. The baseline cannot be represented by a single row in a score-model table.
5. Remove process-history language. The final result is the validated visible-mass baseline, not a "previous" or "frozen" result.

If the fixer cannot produce current machine-readable metadata that makes the visible-mass baseline the final result, then Phase 5 remains blocked. If the fixer wants the optimized score as final, Phase 3/4 regression is required.
