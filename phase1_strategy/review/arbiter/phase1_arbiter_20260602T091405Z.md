## Summary Verdict

**ITERATE.**

The Phase 1 strategy is physically credible and correctly treats the prompt as
a search/template-fit analysis governed by `conventions/search.md`. It covers
the major H to tau tau backgrounds, commits to a simultaneous category fit
with VBF, includes the requested alternative observables, and identifies the
reduced-open-data limitations honestly.

It is not yet a Phase 1 PASS because the critical reviewer identified a real
Category A convention-coverage omission: pileup, named by `search.md` as a
pp-specific replacement systematic, is not explicitly implemented or justified
as inapplicable. Several Category B issues also need correction before PASS
because they are binding-strategy details that later phases must be able to
audit.

## Required Fixes for the Phase 1 Fixer

The fixer must edit `phase1_strategy/outputs/STRATEGY.md` without weakening
existing commitments and must address the items below in priority order:

1. Add an explicit pileup row or pp detector-environment row to the search
   convention enumeration. State "Will implement" with the intended pileup
   profile/weight variation, or "Not applicable because..." with evidence
   that the reduced CMS 2012 files do not support pileup treatment and a
   named coverage/limitation plan that avoids double counting.

2. Define mutually exclusive fit categories. Specify the assignment order
   such as VBF first, then boosted/1-jet, then 0-jet/non-VBF baseline. State
   that any truly inclusive distribution is diagnostic only and is not an
   additional simultaneous-fit category containing duplicated events.

3. Add a statistical-configuration commitment: modified CLs for upper limits,
   one-sided `q0` for discovery with `q0 = 0` when `mu_hat < 0`, one-sided
   limit tests with `mu >= 0`, asymptotic formulae only when bin counts meet
   the convention threshold, and toy validation or direct toys for final
   low-statistics bins.

4. Add quantitative alternative-observable acceptance gates. At minimum bind:
   MVA input `chi2/ndf > 5` requires calibration/removal/justification,
   signal-injection bias must be below the `search.md` 20% threshold, closure
   and GoF must pass for any primary candidate, and a predeclared sensitivity
   improvement is required before replacing `m_vis` as primary.

5. Specify the NN genMET/missing-momentum mass construction: predicted
   quantities, MET magnitude source, four-vector construction, allowed
   reconstructed inputs, train/test/independent-MC validation, data control
   validation, and formal downscope criteria if genMET/truth targets are
   absent or leakage cannot be prevented.

6. Add a region-definition and validation table covering SR, W CR/VR,
   QCD same-sign and anti-isolation CR/VR, top CR/VR, and Z-rich VR. Include
   selection differences from SR, dominant process, subtraction method,
   likelihood/prior usage, transfer factor, closure metric, pass/fail
   threshold, orthogonality rule, and fallback when branches/statistics are
   missing.

7. Strengthen W+jets normalization requirements inside that table or a
   dedicated subsection: report high-mT CR purity after non-W subtraction,
   validate transfer factors separately by fit category where statistics
   allow, and assign/document a larger category-extrapolation uncertainty
   for VBF or other low-statistics transfer factors.

8. Add a binding comparison-target matrix for the final AN. Include this
   analysis's expected/observed `mu`, significance, CLs limit if needed,
   category yields, post-fit signal/background summaries, nuisance impacts,
   and VBF/non-VBF sensitivity; CMS 2014 global and any extractable
   `mu tau_h`/category/control-region comparators; CMS 2018 comparisons with
   energy/luminosity caveats; ATLAS+CMS/PDG/world-average rows where
   meaningful; and explicit "not directly comparable" rows where no clean
   mapping exists.

9. Apply low-cost traceability improvements if time permits: list [D9] in
   the Decisions section or add a decision index, add a commitments-to-phase
   closure table, and add an explicit uncertainty-inflation check such as
   investigating whether the 10-15% DY/tau uncertainties are too broad if all
   relevant validation pulls are below 0.5 sigma.
